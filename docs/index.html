Strategy Overview

This strategy uses Proximal Policy Optimization (PPO) to trade BTC futures based on liquidation map patterns. By analyzing where traders are likely to get liquidated, we anticipate price movements and trade accordingly.

🧠 PPO Reinforcement Learning

Directly optimizes for trading profits rather than prediction accuracy. Learns optimal entry/exit timing through trial and error.

📊 Hybrid CNN + MLP

CNN processes 200 hours of OHLCV candles. MLP handles 34 liquidation map features. Fusion layer combines both for action selection.

📈 Triple-Barrier Labels

+2% profit take, -1% stop loss, 24-hour max horizon. Asymmetric 2:1 risk/reward ratio.

⚡ Go Live Engine

ONNX model deployed in Go for production trading. 100% signal parity verified between Python backtest and Go engine.
