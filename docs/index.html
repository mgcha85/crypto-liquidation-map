Strategy Overview

This strategy uses machine learning to predict price movements based on liquidation map patterns. By analyzing where traders are likely to get liquidated, we can anticipate potential price magnets and resistance zones.

📊 Liquidation Heatmaps

Extract 20+ features from Open Interest data including cluster distances, volume concentrations, and distribution metrics.

🤖 XGBoost Model

Gradient boosting classifier trained on 5+ years of hourly data with class-weighted sampling for imbalanced labels.

📈 Triple-Barrier Labels

Labels generated using +2% profit take, -1% stop loss, and 24-hour horizon for consistent risk/reward.

🔬 Walk-Forward Testing

Out-of-sample testing on 2026 data (4 months) to validate model generalization.

Methodology

1

Data Collection

Download Open Interest (5-min) and OHLCV (1-hour) data from Binance Vision for each symbol. Data ranges from September 2020 to April 2026.

2

Feature Engineering

For each timestamp, compute a 50-hour lookback window to extract liquidation map features (cluster positions, intensities, entropy) and candle features (returns, volatility, ATR).

3

Label Generation

Apply triple-barrier method: +1 if price hits +2% first, -1 if price hits -1% first, within 24-hour horizon. This creates asymmetric risk/reward (2:1).

4

Model Training

Train XGBoost with class weights to handle imbalanced data (~65% sell vs ~35% buy). Hyperparameters: max_depth=4, learning_rate=0.05, n_estimators=150.

5

Backtesting

Test on 2026 data (out-of-sample). Execute trades with 10% position sizing, 0.04% taker fee, 5bps slippage. Compare against Buy & Hold baseline.

Key Findings

What Works

  • Liquidation cluster features are predictive
  • XGBoost outperforms CNN on this dataset
  • All models beat Buy & Hold in bear market
  • BTC model has positive absolute returns

Limitations

  • Test period only 4 months (2026)
  • Bear market bias in test data
  • ETH/SOL have negative absolute returns
  • Win rate ~50% requires good R:R
