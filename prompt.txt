## 프로젝트 개요
## 1. 청산 정보(Event) vs 청산맵(Map)의 차이

직접 만드시기 전에 데이터의 성격을 먼저 구분해야 합니다.

* **청산 정보 (Liquidation Events):** 실제로 터진 포지션의 기록입니다. (사후적 데이터)
* **청산맵 (Liquidation Map):** 특정 가격대에 포지션들이 진입해 있고, 가격이 어디까지 가면 터질지 **예측**한 데이터입니다. (사전적 데이터)

단순히 "어디서 많이 터졌나"를 보시려는 거라면 **청산 정보**만으로 충분하지만, 코인글래스처럼 "어디가 지뢰밭인가"를 보시려면 **Open Interest(미결제약정)** 데이터가 추가로 필요합니다.

---

## 2. 저렴하게(또는 무료로) 히스토리컬 데이터 얻는 법

코인글래스 API 결제 없이 1년 이상의 1시간 단위 데이터를 얻는 가장 좋은 방법은 **거래소의 공공 데이터 센터**를 이용하는 것입니다.

### A. Binance Vision (완전 무료)
바이낸스는 [Binance Vision](https://data.binance.vision/)을 통해 과거 데이터를 CSV/Parquet 형태로 무료 제공합니다.
* **경로:** `data/futures/um/daily/liquidationSnapshot` (청산 이벤트)
* **경로:** `data/futures/um/daily/openInterest` (미결제약정)
* **장점:** 1년 이상의 1시간(또는 더 세밀한) 데이터를 비용 없이 무제한으로 받을 수 있습니다.
* **방법:** Python 스크립트로 해당 경로의 ZIP 파일들을 일괄 다운로드하여 `Polars`나 `DuckDB`로 병합하면 됩니다.

### B. Coinglass Hobbyist ($29)의 한계
Hobbyist 요금제는 **'집계된(Aggregated)'** 데이터만 줍니다. 즉, "1시간 동안 총 100억 터졌다"는 정보는 주지만, "어떤 가격에 들어간 포지션이 터질 위기인가"를 계산하기 위한 개별 틱 데이터는 부족합니다. 따라서 직접 맵을 그리실 거라면 차라리 바이낸스 무료 데이터를 가공하는 게 낫습니다.

## 3. 직접 청산맵을 만드는 알고리즘 (DIY)

데이터 사이언티스트로서 다음 로직을 코드로 구현하시면 코인글래스와 유사한 맵을 만들 수 있습니다.

### Step 1: 데이터 로드
`Open Interest(OI)`와 `Price(OHLCV)` 데이터를 준비합니다.

### Step 2: 포지션 진입 추정 (Accumulation)
가격이 특정 구간에서 횡보하거나 상승할 때 OI가 증가했다면, 해당 가격대에 포지션이 구축된 것으로 가정합니다.
* **가정:** 신규 생성된 OI의 일정 비율을 레버리지 배수별($10x, 25x, 50x, 100x$)로 할당합니다.

### Step 3: 청산가 계산 ($Liq\ Price$)
각 레버리지별로 예상 청산가를 산출합니다.
$$Liq_{long} = EntryPrice \times (1 - \frac{1}{Leverage} + MaintenanceMargin)$$
$$Liq_{short} = EntryPrice \times (1 + \frac{1}{Leverage} - MaintenanceMargin)$$

### Step 4: 맵 업데이트
1.  **가격 이동:** 현재 가격이 위 공식으로 계산된 $Liq\ Price$에 도달하면, 해당 구간의 물량을 삭제(청산 처리)합니다.
2.  **OI 감소:** 실제 시장에서 OI가 감소하면, 누적된 물량에서 비례해서 차감합니다.

### Step 5: 시각화
가격(Y축)과 시간(X축)에 따른 누적 청산 예정 물량을 `Heatmap`으로 시각화합니다. (Python의 `Plotly`나 `Seaborn` 활용)

## 4. 추천 기술 스택 (현업 AI 개발자 맞춤)

사용자의 숙련도를 고려할 때 가장 효율적인 구성입니다.

* **데이터 수집:** `Python` + `bignance-public-data` (GitHub 오픈소스 라이브러리)
* **데이터 처리:** **Polars** (1년치 1h 데이터를 처리할 때 Pandas보다 압도적으로 빠르며, Parquet 포맷 활용 권장)
* **데이터 저장:** **DuckDB** (로컬에서 SQL로 고속 쿼리가 가능하여 시계열 분석에 최적)
* **시각화:** `Plotly` (인터랙티브하게 줌인/아웃하며 특정 가격대 물량을 확인하기 좋음)

- .env.dev, .env.prod, .env.example 를 만들어주면 내가 값을 채울께

https://github.com/ai-boost/awesome-harness-engineering.git 를 참고해서 이 프로젝트에 맞는 구조를 만들어줘.
